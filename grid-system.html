Twenty-four column responsive CSS grid layout

With my earlier intent to go into freelance web development, a great deal of time and effort was put into creating a toolkit that would allow for fast, customizable creation of webpages to a client’s request. While substantial portions of my resources I found open source to implement as necessary, there was not at the time a great grid solution as the modern CSS Grid mechanism. Most layout solutions at the time involved either large amount of screen size queries, followed up by complex to-the-pixel or percent resizing, shoddy looking table schemes, or simply small-screen “mobile first” layouts that would hold massive margins on either side of any decent sized monitor.


Mine was not the first grid system, but perhaps a consequence of the resolutions at the time I was exploring web development. The ones available were largely 8 to 12 columns, mostly in the 960-pixel range. Utilizing the concepts from these grids, I bumped the columns to 24 to allow greater customization, went to 1140 pixels to allow more screen usage, and built the corresponding columns in as percentages rather than pixels. This combined with 2 general queries to optimize smaller screens, and then present the content-blocks in column gives website the ideal mobile responsiveness, while keeping a clean tablet or desktop look that could be applied broadly to sites. The result allows a much more customizable, streamlined, responsive method for page layout design without endless queries or counting pixel by pixel. While the pixel density on newer devices largely dates the relevancy of the mobile query, ultimately it was left alone for people who may still be running lower pixel density web-capable devices.


A demo is implemented here at the bottom of the page. As the webpage is resized, the columns will follow, until the mobile threshold is reached at which time the content will be displayed in a vertical layout fashion, ideal for phone scrolling. While this demo shows the grid system’s full flexibility, this site itself uses the same grid that formats the layout seen.

Some ways to break vertical flow on the page include adding vertical margin or padding to the specific rows or items. Simple break tags implemented in the html can do this as well.
Items specific to a column may be placed in similar width columns of different rows, while other content meant to be related may be placed in the same column of the same row.

The freedom to have vertical breaks or allow rows directly adjacent atop one another allows content to be displayed in whichever manner is ideal for whichever desired layout. Because the horizontal aspects are handled by the grid system, it is best to leave these to the template itself, while merely utilizing vertical top and bottom margin and/or padding adjustments to create the desired effects. This helps with maintaining similar layouts whether in a full desktop view, or in a slim small-screen viewer when the content stacks atop one another.
